2. STATEFUL HYSTERESIS (commit 4376a13 + 85bde7d)
═══════════════════════════════════════════════════════════════════════════════

Asymmetric entry vs exit thresholds in signals/generator.py.

Threshold design (data-driven from May 3 distribution audit):
  ENTRY: {h=1: 0.80, h=3: 0.60, h=5: 0.60}
  EXIT:  {h=1: 0.65, h=3: 0.50, h=5: 0.50}

Day-over-day prob_up noise structure (audited):
  median |change| = 0.046
  90th percentile = 0.128
  95th percentile = 0.160

Gap rationale: 0.10 entry-exit gap (h=3, h=5) ≈ 2x median noise, filters most
random walk. 0.15 gap (h=1) ≈ 3x median noise, appropriate for noisier
short-horizon model.

Backtest on 9,411 historical predictions:
  h=3: signal flips 641 -> 249 (-61%), BUY days 1336 -> 1235 (-7.6%)
  h=5: signal flips 565 -> 242 (-57%), BUY days 1339 -> 1244 (-7.1%)
  h=1: minimal flip change (24 -> 22) but BUY days 13 -> 26 (extended
       hold-through after rare 0.80 entries — correct hysteresis behavior)

Implementation: new helper _lookup_yesterday_signal(ticker, horizon, as_of)
queries accuracy.db.predictions (read-only) for prior signal. Edge cases
(no DB, no prior, error) return None which caller treats as HOLD.

Plus scripts/threshold_review.py (commit 85bde7d) — re-runnable monthly audit
of prob_up distribution + signal frequency at each threshold + current
ENTRY/EXIT constants for review.

3. POINT-IN-TIME HONEST WALK-FORWARD (commits 7b6f3e9 + 39a4733)
═══════════════════════════════════════════════════════════════════════════════

The big one. WorldQuant principle (Finding Alphas Ch. 8): walk-forward
backtests are worse than useless if they have leaks. Old walk_forward.py
read from prediction_features table (~27 cols) — fast but scope-limited.

Real audit found 3 leak vectors in stored-data feature loaders that needed
patches:

  PATCH 1: load_sentiment_scores — added as_of parameter that filters
           by created_at. sentiment_scores already had created_at column
           (good).

  PATCH 2: insider_flows table needed schema migration — NO created_at
           column existed. Added column, backfilled 2,580 historical rows
           with γ logic (trade_date + 2 business days, matching SEC Form 4
           filing rule). Updated _upsert_to_db to stamp created_at at
           insert time. Updated _init_db CREATE TABLE for fresh installs.

  PATCH 3: _load_insider in builder.py — added as_of parameter, branches
           query: with as_of -> WHERE created_at <= ?, without -> original.

  PATCH 4: load_earnings_features — added as_of parameter, filters both
           earnings_surprises (created_at) and earnings_calendar (updated_at).

  PATCH 5: builder.py wiring — passes end_date as as_of to all PIT-aware
           loaders. Critical: when training_mode=True, skips LIVE API calls
           (load_uw_earnings_features, _load_insider_uw) entirely and routes
           to PIT-filtered SQLite paths.

  PATCH 6: analysis/walk_forward.py — new function load_panel_pit() that
           calls build_feature_dataframe(end_date=prediction_date,
           training_mode=True) for each unique (ticker, prediction_date),
           takes the last row as features as-of that date. New CLI flags
           --pit and --limit. Backward compat: load_panel without --pit
           still works.
